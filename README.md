# Flow Memory AI

The Human Compute Network: local-first AI memory, signed receipts, and verifier-first agent infrastructure.

## Built today — Jun 17, 2026

- **Receipted Memory**: Chrome/Edge MV3 extension for encrypted local memory fields, Claude/ChatGPT release gates, P-256 signed disclosure receipts, hash-chained receipt log, compliance ZIP export, and bundled offline verifier.
- **Receipted Memory verifier**: zero-runtime-dependency TypeScript verifier matching the Python/OpenSSL P-256 receipt vectors, plus standalone offline HTML verifier and sample receipt.
- **Mandate Companion**: MV3 extension for local mandate signing, browser-action receipts, executor gating, encrypted BYO provider-key storage, manual no-provider demo path, revocation records, redacted export, and offline verifier.
- **Release docs**: store listing copy, privacy policy, terms, support path, first-run handoff, security review, and live-evaluation notes.

Primary repo: [FlowmemoryAI/flow-memory](https://github.com/FlowmemoryAI/flow-memory)

Latest public update: Jun 17, 2026 — Receipted Memory and Mandate Companion launch work is pushed in `flow-memory`.

## Honesty boundary

Flow Memory receipts prove signed local evidence and tamper checks. They do not make providers blind, private, legally compliant, or risk-free. Released data is visible to the provider or website the user sends it to.
